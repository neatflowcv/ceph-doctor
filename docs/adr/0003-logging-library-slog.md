# ADR 0003: 로깅 라이브러리 - slog

날짜: 2026-02-23
상태: 채택

## 배경

프로젝트 전반에서 구조화 로그를 일관되게 출력할 수 있는 로깅
라이브러리가 필요하다. 요구사항:

- 표준에 가까운 API로 유지보수 부담이 적을 것.
- 구조화 로그(키-값)를 기본으로 지원할 것.
- 프로젝트 전반에서 동일한 로깅 인터페이스를 사용할 수 있을 것.

## 결정

로깅 라이브러리로 Go 표준 라이브러리 `log/slog`를 사용한다.

## 근거

- 표준 라이브러리이므로 장기적으로 안정적이고 외부 의존성을
  최소화할 수 있다.
- 구조화 로그를 기본으로 지원하며 커스텀 핸들러로 확장이 가능하다.
- 생태계 도입 비용이 낮고 구성/테스트가 단순하다.

## 대안

- zap: 고성능 구조화 로깅을 제공하지만 외부 의존성 추가가 필요하다.
- zerolog: JSON 중심의 고성능 로깅을 제공하지만 표준 라이브러리는 아니다.
- logrus: 널리 사용되지만 유지보수 모드이며 신규 기능은 중단되었다.

## 결과

- 애플리케이션의 로깅은 `log/slog` 기반으로 구현한다.
- 필요 시 핸들러를 교체해 출력 포맷을 조정한다.
